# Slice: extract-compiler

## Delivers
The `SchemaCompiler` engine library + its CLI/skill wiring + the correctness anchor (the whole
extraction, minus the Drift dedupe which is the downstream slice).

- A pure `SchemaCompiler` type in `Sources/AISDDEngine/SchemaCompiler.swift`: parsed `SchemaSpec`
  (+ name/version) → compiled `CheckSpec(s)`, each tagged by an `Origin` (`autoGenerated` vs
  `authored`). No IO in the core.
- **Tier-1 (structural):** the fixed mechanical template keyed off schema name/version/format → one
  `<name>.structure` deterministic check, `required: true`. MUST reproduce the committed gate (C5).
  Command form matches the COMMITTED reality `swift run ai-sdd check <schema-path> <artifact-path>`
  (C3), not the skill's abbreviated `ai-sdd check …`.
- **Tier-2 (rules) — mechanical part (C1):** add optional `rules` to `SchemaSpec` (`AISDDModels`); an
  explicit-`command` rule compiles verbatim to a deterministic check, blocking iff its severity is
  blocking (auto-generated).
- **Tier-3 (judge) + intent rules — the boundary (C1):** add optional `judge` to `SchemaSpec`; emit
  judges (and intent-only rules) as explicit `authored` markers (advisory, no fabricated command /
  verdict). Never faked.
- A thin read-only `ai-sdd compile <schema>` `ParsableCommand` in `Sources/AISDDCLI/main.swift`
  (registered in `subcommands:`): prints each compiled check as committed-shape `kind: Check` YAML,
  tagged by origin; `--json` for machine output; `--help` documents it. Mirrors `check`/`scope`/`cover`.
- Centralize new path literals in `Layout.swift` (schema source path, versioned artifact path, check
  source path); render committed-shape YAML at an engine edge (keep the compiler core encoder-free).
- Update the `ai-sdd-compile-schema` skill to drive `ai-sdd compile` for the deterministic tiers
  (auto-generated checks committed as-is; authored markers finished + eval-gated by hand).
- Tests in `Tests/AISDDEngineTests/`: the **anchor** (compile every real `.ai-sdd/schemas/*.schema.yaml`
  and assert the structural check == the committed `checks/<name>.structure.check.yaml`), plus per-tier
  units (structural template, explicit-command rule blocking/advisory, intent→authored, judge→authored,
  tier ordering, YAML round-trip).

## Acceptance
- `SchemaCompiler.compile(...)` over each committed schema reproduces the committed structural check
  (same name, checkKind, command, required) — verified against the real repo.
- Explicit-command rule → verbatim deterministic check, blocking by severity; intent-only rule and
  judge → advisory `authored` markers (no fabricated command).
- `ai-sdd compile <schema>` prints the compiled checks (and `--json`); `--help` documents it.
- The `ai-sdd-compile-schema` skill references the engine command for the deterministic tiers.
- `swift build`, `swift test`, `swift run ai-sdd validate .ai-sdd` green.
